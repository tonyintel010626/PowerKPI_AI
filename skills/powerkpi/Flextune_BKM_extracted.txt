INTRO TO FLEXTUNE 

1.1  What is Flextune? 

Flextune is an automation utility designed to simplify workload execution by leveraging 
Hopper commands in a controlled and standardized way. 

Instead of manually copying, pasting, and modifying Hopper commands for each 
workload, Flextune centralizes and automates this process, significantly reducing 
operational overhead. 

Why Flextune is Used 

Traditionally, running workloads requires: 

ď‚·  Manual pasting of Hopper commands/ Using Pacs 
ď‚·  Repeated variable changes for different workloads 
ď‚·  High risk of human error during execution 

Flextune eliminates these inefficiencies by handling command execution and variable 
management automatically. 

(cid:33158)(cid:57393) Key Benefit: Engineers no longer need to interact directly with Hopper commands 
for routine workload execution. 

Key Objectives of Flextune 

ď‚·  Automate Hopper command execution 
ď‚·  Eliminate repetitive manual power capture 
ď‚·  Standardize workload execution across Power Stations 
ď‚· 
ď‚·  Allow engineers to focus on debugging and analysis instead of operations 

Improve efficiency in data capturing

Workload Capture Types 

There are three workload types, based on the capture method. 

1.workload_daq 

Normal power capture using FlexLogger. 

2.workload_soc 

SoCWatch capture only. 

No power (DAQ) data is collected. 

3.workload_daq_soc 

Parallel capture of both power (FlexLogger) and SoCWatch data. 

Used when both power and SoC information are required. 

Workflow Selection and Execution Priority 

Flextune executes workloads in the exact order they are selected in the workload list. 

The execution priority is determined by the selection sequence, not by workload name or type. 

Example 

If the following workloads are selected in this order: 

ď‚· 

cms_daq 

ď‚·  busy_idle_daq 

ď‚·  window_opt 

Flextune will execute them in the following sequence: 

1.  cms_daq 

2.  busy_idle_daq 

3.  window_opt

Finding and Managing Output Files 

After workload execution is completed, the output files are automatically generated and saved 

by Flextune. 

Output File Location 

All output files are located at the following directory: 

C:\tuning_workload 

Each execution generates an output folder or file that contains all workloads selected and run in 

that session. 

17 

 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
Output File Naming Logic 

The output file or folder name is automatically created based on: 

ď‚·  Execution sequence 

ď‚·  Time of execution 

Because the naming is time- and sequence-based, the file name may not clearly reflect the 

workload content. 

Important Recommendation 

It is strongly recommended to rename the output file or folder immediately after workload 

completion to avoid confusion during later analysis. 

All workloads selected in a single run will be saved together in the same output folder. 

Example Scenario 

A user has two data-capture tasks on the same day. 

Morning run: 

ď‚·  Workloads executed: Netflix and IDON 

ď‚·  Result: One output folder containing both Netflix and IDON data 

Afternoon run: 

ď‚·  Workloads executed: YouTube and Catapult browsing 

ď‚·  Result: One output folder containing both YouTube and Catapult browsing data 

In this case, two separate output folders will be created: 

ď‚·  One for the morning run (Netflix + IDON) 

ď‚·  One for the afternoon run (YouTube + Catapult browsing) 

Renaming the folders after each run helps clearly distinguish between different scenarios.
