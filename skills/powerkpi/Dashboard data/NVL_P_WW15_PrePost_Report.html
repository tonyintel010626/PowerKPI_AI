5. HSDES Sighting Analysis — Why Power is Not Meeting Projections

Source: YC_Debugger agent HSDES search — 25,042 open sightings scanned → 2,025 NVL hits → 209 power/PM-related → 28 directly relevant. Analysis performed WW15 2026.

Key Finding: The NVL-P post-silicon power gap is NOT a single root cause — it is an overlay of at least 6 independent issues across VNNAON, VCCSA, VCCCORE, PkgC residency, CNVi, and fuse/measurement infrastructure. Most have open sightings on NVL-S/NVL-Hx variants. One critical sighting (HSD 15019229805) is NVL-P specific.

5.2 CMS (25°C) — 136.83 mW vs 48 mW projected (2.85×)

Same root causes as IDON 5.1.1 (sightings 15018538099, 22021310430, 14027432226, 14027518830, 14027573978). CMS has the worst ratio (2.85×) because it is the lightest workload — always-on/leakage components dominate the 48 mW pre-silicon target more than in heavier workloads. Absolute mW values on VNNAON may be similar to IDON but represent a larger fraction.

Additional Debug: Characterise CMS workload CPU activity — is it truly idle or are there periodic wake events? Check PkgC residency during CMS — if not entering deep PkgC states, the gap will compound.

5.4 Catapult @40°C — 1521.29 mW vs 1120 mW projected (1.36×)

P_VCCSA is the primary contributor (1.51×). Same root causes as YouTube section 5.3.2: sightings 14026486014, 22021638359, 14027115826. Catapult is a heavier workload so the VCCSA gap is proportionally smaller (1.51× vs 1.77×) because active power dominates over idle leakage.

Note: No documented sighting found specifically for Catapult workload on NVL-P. Recommend filing new sighting if Catapult power gap persists after VCCSA pcode fix (HSD 22021638359) and GT at1 fix (HSD 14027115826) are applied.
